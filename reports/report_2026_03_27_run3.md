## Recommendations

### P0 — Fix Margin Clamp Leverage Bug

**Problem**: The margin clamp check uses `leverage=1` instead of the configured leverage. Position size is clamped to 1/3 of intended.

**Fix**: Pass the correct leverage (3x) to the margin clamp calculation. The log shows `Paper open SHORT SOLUSDT: qty=120.34 margin=3333.73 leverage=3x` — the leverage IS available, it's just not being used in the clamp check.

### P1 — Fix Analyst Structured Output Consistency

**Problem**: The Analyst outputs `setup_valid=False` when its own reasoning says the setup is valid (TAOUSDT conf=8 at 13:09:51). This is either a structured output parsing issue or the model contradicting itself.

**Options**:
- Add a post-processing check: if `confluence >= threshold AND reasoning contains "valid"`, override to `setup_valid=True`
- Add explicit instruction in the prompt: "If your calculated confluence score meets or exceeds the threshold of 7, you MUST set setup_valid to True"
- Use Pydantic validation: if `confluence >= 7`, force `setup_valid=True`

**Recommended**: Enforce in the prompt AND add a Pydantic validator as a safety net. The Analyst should not be able to score confluence=8 and then say "not valid."

### P2 — Add Scout Deduplication / Cooldown

**Problem**: SOLUSDT was sent to the Analyst 16 times — 5 times after the trade was already open. XRP was sent 12 times with no material data change.

**Fix**: After a symbol reaches the Analyst (or Trader), apply a cooldown of N cycles (e.g., 3–5) before the Scout can flag it again. This would reduce wasted Analyst calls by ~60%.

**Alternative**: If a symbol already has an open position, skip the pipeline entirely for that symbol.

### P3 — Add "Post-Selloff Bounce" Regime Detection

**Problem**: The system entered a SHORT 2+ hours after a selloff exhausted itself. All 8 non-traded symbols bounced during the session. The market was clearly in recovery mode.

**Fix**: Add a higher-level regime detector that identifies "post-selloff bounce" conditions:
- RSI < 35 across multiple symbols
- 2+ hours since the initial impulse
- Volume declining from selloff peak
- MACD histogram turning positive

When detected, increase the confluence threshold to 9 or add a mandatory "wait for retest" condition.

### P4 — Improve TP Target Selection

**Problem**: SOL's TP at $82.53 was 0.70% below entry, but the actual intraday low was $82.96 (only 0.18% below entry). The TP was unreachable.

**Fix**: Use ATR-based TP targets with awareness of recent lows. If the TP is below the recent swing low, it's unrealistic. The Trader should calculate TP as `min(ATR-based target, recent_swing_low - buffer)`.

### P5 — Reduce Trader Latency (Carried from Run 2)

**Problem**: Trader p50=38s, max=43s. While staleness is no longer blocking trades (threshold now 120s), the Trader still takes 8–9x longer than target.

### P6 — Calibrate Staleness Threshold

**Problem**: The threshold was increased from 30s to 120s — now decisions up to 2 minutes old can execute. This is too permissive for a 1m timeframe.

**Recommended**: 60s is the right balance. At current Trader latency (38–43s) + pipeline overhead (~10s), most decisions are 48–53s old. A 60s threshold would pass these while still rejecting truly stale decisions.
